Unversioned or Mutable Action Reference
.github/workflows/ci.yml · line 27
HIGH
What
Action `actions/checkout@main` uses mutable branch reference '@main'. If the upstream action is compromised or its history is rewritten, your workflow will silently execute malicious code.
Evidence
actions/checkout@main
Fix
Pin to an immutable reference. Recommended fix: uses: actions/checkout@ # find the SHA on the action's releases page Use a tool like `pinact` or Dependabot to keep pins up-to-date.

SEC-003
Missing Top-Level Permissions Block
.github/workflows/ci.yml
MEDIUM
What
No `permissions:` block is defined at the workflow level. Without explicit permissions, GitHub uses the repository default (often `write` to contents), which may be broader than needed.
Fix
Add a top-level permissions block with the minimum required scopes: permissions: contents: read # add only the scopes your workflow actually needs, e.g.: # pull-requests: write # to post PR comments # packages: write # to push to GHCR # id-token: write # for OIDC-based cloud auth
